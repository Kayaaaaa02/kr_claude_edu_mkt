오늘 워크숍의 핵심 메시지는 한 문장으로 정리할 수 있습니다.

오늘 중요한 것은 코드를 많이 아는 것이 아닙니다. 업무 목적을 명확하게 정의하고, AI가 그 목적에 맞는 결과물을 만들도록 지시하는 방식입니다.

## 5. Use Case와 Claude Code 핵심 개념

**페이지 영역:** Chapter 01  
**HTML 기준:** `<section class="chapter-section" id="ch1">`  
**화면 표시:** “결과물 먼저, 개념은 짧게.”

이제 왜 이런 방식이 필요한지 실제 사례를 통해 보겠습니다.

AI 도구를 배울 때 개념부터 길게 들어가면 실제 업무와 연결되기 어렵습니다. 그래서 오늘은 결과물을 먼저 보고, 그 결과물을 가능하게 하는 개념을 짧게 정리하겠습니다.

### 5-1. F&F 한국 사례

**페이지 영역:** Chapter 01 / 1-1  
**HTML 기준:** `#ch1` 내부 `F&F 한국 사례`

F&F 한국에서는 이미 HTML 기반 보고서나 대시보드 형태의 결과물을 업무에 활용하고 있습니다.

기존에는 Excel에서 데이터를 정리하고, PPT로 복사하고, 다시 디자인을 다듬는 방식이 일반적이었습니다. 하지만 이 방식은 반복 작업이 많고, 데이터가 바뀔 때마다 보고서를 다시 만들어야 합니다.

HTML 기반 대시보드는 이 흐름을 바꿉니다. 데이터, 화면, 보고서가 하나의 구조로 연결될 수 있고, 같은 패턴을 반복 업무에 재사용할 수 있습니다.

화면의 예시 링크는 이미 만들어진 보고 자료입니다. 이런 형태를 오늘 우리가 직접 만들어보는 것이 목표입니다.

### 5-2. Claude Code의 확장 메커니즘

**페이지 영역:** Chapter 01 / 1-2  
**HTML 기준:** `#ch1` 내부 `Claude Code의 확장 메커니즘`

Claude Code에는 여러 확장 메커니즘이 있습니다.

오늘 가장 중요하게 볼 것은 Skills입니다.

Skills는 Claude가 특정 작업을 더 잘 수행하도록 만드는 지침서입니다. 예를 들어 대시보드 UI를 만들 때 어떤 디자인 원칙을 적용할지, PPT를 만들 때 어떤 구조를 따라야 할지 Claude에게 알려주는 매뉴얼이라고 이해하시면 됩니다.

Plugins는 여러 Skills나 도구를 묶어서 한 번에 설치하는 패키지입니다. 오늘은 실습 환경 세팅에 사용합니다.

Memory는 Claude가 프로젝트나 조직의 선호를 기억하도록 돕는 기능입니다. MCP는 외부 시스템과 연결하는 방식입니다. Hooks는 특정 작업 전후에 자동으로 실행되는 트리거입니다.

오늘은 모든 기능을 깊게 다루기보다는, Skills와 Plugins를 실제로 사용해보고 나머지는 이후 자동화 단계에서 활용할 개념으로 이해하겠습니다.

## 6. PLAN 프레임워크

**페이지 영역:** Chapter 02  
**HTML 기준:** `<section class="chapter-section" id="ch2">`  
**화면 표시:** “본격적인 시작에 앞서 5분 계획이 전체의 완성도를 좌우합니다.”

이제 오늘 워크숍에서 가장 중요한 설계 단계입니다.

AI로 결과물을 만들 때 가장 흔한 실수는 바로 구현부터 요청하는 것입니다. 예를 들어 “매출 대시보드 만들어줘”라고만 하면 화면은 만들어질 수 있습니다. 하지만 그 화면이 누구를 위한 것인지, 어떤 의사결정을 지원하는지, 어떤 데이터가 중요한지 반영되지 않을 가능성이 높습니다.

그래서 오늘은 먼저 PLAN을 작성합니다.

이 PLAN은 단순한 사전 메모가 아니라, 결과물의 품질을 결정하는 업무 설계서입니다.

### 6-1. 5단계 PLAN 워크시트

**페이지 영역:** Chapter 02 / 5단계 PLAN 워크시트  
**HTML 기준:** `#ch2` 내부 `5단계 PLAN 워크시트`

PLAN은 다섯 가지 질문으로 구성됩니다.

첫 번째는 목적입니다. 누가, 언제, 어떤 의사결정을 위해 이 화면을 보는지 정의합니다.

두 번째는 데이터입니다. 어떤 데이터를 사용할 것인지, 어디서 가져올 것인지, 파일 형식은 무엇인지 확인합니다.

세 번째는 화면 스케치입니다. 한 화면에 반드시 들어가야 할 핵심 지표를 3개에서 5개 정도로 정리합니다.

네 번째는 보고서 형식입니다. 최종 결과물이 HTML인지, PPT인지, Excel인지 정합니다.

다섯 번째는 배포 방식입니다. 본인 PC에서만 볼 것인지, URL로 공유할 것인지, 사내 시스템에 올릴 것인지 결정합니다.

이 다섯 가지가 정리되면 Claude Code가 해야 할 일이 훨씬 명확해집니다.

### 6-2. Plan Mode

**페이지 영역:** Chapter 02 / 2-1  
**HTML 기준:** `#ch2` 내부 `plan 짜는 방법 — Plan Mode`

이때 사용하는 기능이 Plan Mode입니다.

Plan Mode를 켜면 Claude가 바로 코드를 작성하지 않습니다. 먼저 작업 계획을 보여주고, 사용자가 승인해야 실제 구현을 시작합니다.

입문자에게 특히 유용한 이유는 작업 방향을 중간에 확인할 수 있기 때문입니다. AI가 잘못된 방향으로 파일을 만들기 전에, 설계와 접근 방식을 먼저 검토할 수 있습니다.

오늘 실습에서는 가능한 한 Plan Mode를 켠 상태에서 진행하겠습니다.

VS Code 확장 프로그램에서는 `Shift + Tab`을 두 번 눌러 Plan Mode로 진입할 수 있습니다. 터미널에서는 `/plan`이라고 입력하면 모드를 전환할 수 있습니다.

## 7. 추천 시나리오

**페이지 영역:** Scenario Section  
**HTML 기준:** `<section class="block" id="scenarios">`  
**화면 표시:** “추천 시나리오, 이대로 따라가며 핸즈온.”

오늘의 추천 시나리오는 Tmall과 Douyin의 주간 매출 대시보드입니다.

이 시나리오는 패션 업계에서 자주 발생하는 업무 흐름과 연결됩니다. 채널별 매출을 확인하고, 전주 대비 성과를 보고, 어떤 상품과 카테고리가 성과를 만들었는지 빠르게 파악하는 것이 목적입니다.

샘플 데이터는 `sample_data.xlsx`를 사용합니다. 실제 업무 데이터를 사용하실 경우에는 지식그래프나 사내 데이터 소스에서 매출 데이터를 내려받아 같은 구조로 적용할 수 있습니다.

핵심 지표는 네 가지입니다.

첫째, 전주 대비 증감률입니다.

둘째, 매출 기준 Top10 상품입니다.

셋째, 카테고리와 채널을 교차해서 보는 히트맵입니다.

넷째, 채널별 주간 매출 흐름입니다.

이 지표들은 단순히 보기 좋은 차트가 아니라, 매주 월요일 채널 ROI를 점검하기 위한 의사결정 지표입니다.

### 7-1. 예시 프롬프트

**페이지 영역:** Scenario Section / 예시 프롬프트  
**HTML 기준:** `#scenarios` 내부 `예시 프롬프트 — Plan Mode로 계획부터`

프롬프트를 작성할 때는 “대시보드를 만들어줘”로 끝내지 않습니다.

먼저 목적을 말합니다. 매주 월요일 채널 ROI를 점검하기 위한 대시보드라고 정의합니다.

그다음 핵심 지표를 지정합니다. 채널별 주간 매출, 전주 대비 증감률, Top10 상품, 카테고리×채널 히트맵을 포함합니다.

차트 타입도 후보를 줍니다. 막대, 라인, 히트맵처럼 화면 구성에 필요한 시각화 방식을 지정합니다.

마지막으로 스타일 방향을 명확히 합니다. “AI 기본 스타일 금지, 미니멀”처럼 피해야 할 방향과 원하는 방향을 함께 적어줍니다.

이렇게 요청하면 Claude가 먼저 설계안을 제시하고, 우리가 OK라고 승인한 뒤 실제 구현을 진행하게 됩니다.

## 8. Claude Code 핵심 개념 Deep-Dive

**페이지 영역:** Concepts Deep-Dive  
**HTML 기준:** `<section class="chapter-section" id="ch2b">`  
**화면 표시:** “클러드 코드의 핵심 기능 5가지를 쉽게 풀어보면.”

본격적인 실습 전에 핵심 개념을 한 번 더 정리하겠습니다.

처음 들으면 용어가 조금 낯설 수 있습니다. 하지만 실무 관점에서는 각각의 역할을 단순하게 이해하면 됩니다.

Skills는 업무 매뉴얼입니다. 특정 작업을 할 때 어떤 방식으로 접근해야 하는지 Claude에게 알려주는 지침입니다.

Plugins는 스킬 묶음 설치입니다. 여러 도구를 하나의 패키지처럼 설치해 사용할 수 있습니다.

MCP는 외부 시스템 연결선입니다. Claude Code가 사내 시스템, 데이터베이스, Slack, Figma 같은 도구와 연결될 수 있도록 하는 표준 방식입니다.

오늘은 Skills와 Plugins를 직접 사용하고, Memory, Hooks, MCP는 다음 단계 자동화를 위한 개념으로 이해하겠습니다.
