전체 원칙: “보고서를 만드는 것”이 아니라 “보고서 생산라인을 운영하는 것”

이 프로세스의 핵심은, 직원들이 매번 새롭게 고민해서 예쁜 보고서를 만드는 것이 아닙니다. 
입력값을 구조화하고, AI를 각 단계에 맞게 배치해서, 일정한 품질의 대표님 보고서를 반복 생산하는 것입니다.

즉 앞으로는 아래처럼 바뀌어야 합니다.
과거 방식: 사람이 처음부터 끝까지 다 만듦
바뀔 방식: 사람이 방향과 판단을 잡고, AI가 구조화·문안화·시각화·산출을 분담

여기서 가장 중요한 원칙은 3가지입니다.

첫째, 사람과 AI의 역할을 명확히 나눈다
사람은 의사결정, 우선순위, 숫자 검증, 최종 판단을 담당하고,
AI는 구조화, 초안 작성, 표현 정리, 시각화 보조, HTML 산출 보조를 담당합니다.
AI에게 “좋은 보고서를 알아서 만들어라”라고 하면 실패합니다.
반드시 어떤 단계에서 어떤 역할만 맡길지를 정해야 합니다.

둘째, 초안 단계와 산출 단계는 분리한다
전략이 아직 안 굳었는데 HTML부터 만들면, 뒤에서 수정이 폭증합니다.
따라서 전략 구조 확정 → 문안 확정 → 비주얼 확정 → HTML 산출 순서를 지켜야 합니다.

셋째, AI 워싱은 마지막 단계에서 1회만 한다
AI 톤으로 보이게 만들겠다고 초반부터 계속 손대면,
문장은 좋아 보여도 메시지가 흐려지고, 직원 리소스만 늘어납니다
